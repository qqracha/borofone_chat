# Тесты

## Тесты

Тесты нужны, чтобы гарантировать идемпотентность в enforce-режиме, поведение без enforce, а также поведение при конкурентных запросах.&#x20;

### Запуск

Обычно достаточно:

* `pytest -q`&#x20;

Более подробно:

* `python -m pytest -s -vv`&#x20;

С логами:

* `python -m pytest -vv -s --log-cli-level=INFO`

### Что проверяем

Минимальный набор сценариев:

* `enforce_nonce=true` и одинаковый `nonce` -> возвращается один и тот же `id` (dedupe).
* `enforce_nonce=false` и одинаковый `nonce` -> создаются разные сообщения.&#x20;
* `enforce_nonce=true` без `nonce` -> ошибка валидации, 409 error (контракт).&#x20;
* Конкурентные запросы с одинаковым `nonce` -> максимум одна запись в БД в пределах TTL-окна.&#x20;

### Очистка после тестов

Для Postgres быстрее всего чистить таблицы через:

&#x20;`TRUNCATE ... RESTART IDENTITY CASCADE`.&#x20;

Для Redis лучше удалять ключи по шаблону через SCAN, а не KEYS, чтобы не блокировать сервер на больших наборах ключей.&#x20;
